🔹 WEEK 2 PROGRESS REPORT (Backend Setup & Database Design Phase)
Review Date: ___________

Details of Work Done and Review of Meeting:
--------------------------------------------------------------------------------
1. Set up the backend development environment using Node.js and Express.js with 
   CORS, security headers (Helmet), and rate limiting.

2. Designed and implemented database schemas for core entities:
   - User model: name, email, password, year, department, role, skills, interests, CGPA
   - Mentorship model: mentorId, menteeId, status (Pending/Accepted/Rejected/Terminated)

   scalability and future analytics integration.

   support and TLS options.

7. Implemented route structure: /api/auth, /api/users, /api/mentorship,

🔹 WEEK 3 PROGRESS REPORT (Authentication & Access Control Phase)
Review Date: ___________

Details of Work Done and Review of Meeting:
--------------------------------------------------------------------------------
1. Implemented user authentication using JWT (jsonwebtoken) with configurable 
   secret and expiration (JWT_EXPIRE=7d).

2. Integrated institute email-based registration with domain validation 
   (ALLOWED_DOMAINS) to ensure verified and institute-exclusive access.

3. Implemented secure password handling using bcrypt hashing (User model 
   pre-save hook and comparePassword method).

4. Designed and implemented role-based access control (RBAC) middleware:
   - checkRole(...roles) for granular permission control
   - isMentor / isMentee for mentorship-specific routes
   - Applied to users, mentorship, interactions, and discussions routes

5. Implemented login flow (POST /api/auth/login) with email/password validation 
   and token generation.

6. Implemented register flow (POST /api/auth/register) with role, year, and 
   email domain validation.

7. Implemented protected routes using verifyToken middleware; unauthorized 
   requests return 401.

8. Verified authentication flows through API testing dashboard at 
   http://localhost:5000 (Register, Login, and protected endpoint testing).

9. Discussed security considerations: token expiration, protected routes, 
   unauthorized access prevention, and input validation (express-validator).

10. Conducted a progress review meeting and received feedback on security 
    and session handling.
